# Опис функціоналу (Functionality)

Система MiroFish складається з кількох ключових модулів, кожен з яких відповідає за певний етап симуляції та аналізу.

## 1. Модуль обробки документів (Data Ingestion)
Цей модуль відповідає за прийом неструктурованих даних (звітів, новин, статей) від користувача.
- **Підтримувані формати:** PDF, Markdown (.md), звичайний текст (.txt).
- **Функціонал:** Текст очищується, розбивається на фрагменти (chunking) та передається до підсистеми графа знань.

## 2. Модуль генерації онтології (Ontology Generation)
Перш ніж будувати граф знань, система повинна зрозуміти, які саме сутності та зв'язки важливі для конкретної задачі.
- LLM аналізує текстові документи та формує "Насіння реальності" (Reality Seed).
- Автоматично створюється онтологія: визначаються типи сутностей (Особа, Організація, Подія, Продукт) та можливі типи їхніх взаємозв'язків.

## 3. Модуль оркестрації агентів (Agent Orchestration)
Серце системи, що відповідає за створення та управління ШІ-персонами.
- **Генерація персон (Personas):** На основі графа знань модуль створює сотні унікальних профілів агентів. Кожному призначається:
  - Демографія (вік, стать, професія).
  - Психотип за MBTI (наприклад, INTJ, ESFP), що впливає на тон повідомлень.
  - Унікальна пам'ять (що агент знає про події зі "світу").
- **Управління часом:** Модуль визначає "робочі" та "пікові" години активності для різних груп агентів.

## 4. Двоплатформна симуляція (Dual-Platform Simulation)
MiroFish імітує сучасний інформаційний простір через дві платформи:
- **Платформа 1 (Стрічка):** Алгоритми заохочують швидке поширення коротких повідомлень (вірусність).
- **Платформа 2 (Спільнота):** Алгоритми створюють "інформаційні бульбашки" (Echo Chambers), де агенти взаємодіють у глибоких гілках обговорень.

## 5. Модуль аналітики та звітності (Report Agent)
Автономний агент-аналітик, який підбиває підсумки симуляції.
Він використовує наступні інструменти (Tools):
- **InsightForge:** Глибока атрибуція. Зіставляє дані реального світу зі станом симуляції.
- **PanoramaSearch:** BFS-пошук по графу для відстеження шляхів поширення інформації.
- **QuickSearch:** Швидкий пошук фактів по базі GraphRAG.
- **InterviewSubAgent:** Модуль віртуального інтерв'ю для автоматичного опитування агентів щодо їхніх поглядів.

## 6. Глибока взаємодія (Deep Interaction)
UI-модуль, що дозволяє користувачеві в режимі реального часу:
- Спілкуватися в чаті з будь-яким симульованим агентом.
- Спілкуватися з Report Agent для уточнення даних зі звіту.
- Надсилати масові опитування (Surveys) вибірці агентів для отримання статистичних даних (наприклад, NPS).
