# Посібник з розгортання (Deployment Guide)

Цей документ містить інструкції щодо розгортання системи MiroFish у різних середовищах.

## 1. Локальне середовище (Development)

Для розробки та тестування на локальній машині:

### Вимоги
- Node.js (18+)
- Python (3.11+)
- менеджер пакетів `uv`

### Інсталяція
1. Клонування репозиторію.
2. Встановлення залежностей:
   ```bash
   npm run setup:all
   ```
3. Копіювання `.env.example` у `.env` та налаштування змінних (докладніше у `CONFIG.md`).

### Запуск
```bash
npm run dev
```
Фронтенд буде доступний на порту `3000`, бекенд на порту `5001`.

## 2. Розгортання за допомогою Docker Compose (Production-ready)

Для швидкого розгортання в ізольованому середовищі:

### Вимоги
- Docker Engine
- Docker Compose

### Кроки
1. Налаштуйте файл `.env`.
2. Запустіть контейнери:
   ```bash
   docker compose up -d
   ```
3. Щоб зупинити систему:
   ```bash
   docker compose down
   ```

## 3. Production розгортання (Nginx + PM2/Gunicorn)

Для забезпечення високої доступності та продуктивності рекомендується використовувати наступну конфігурацію:

### Frontend
- Побудова статичних файлів:
  ```bash
  npm run build
  ```
- Розгортання статики через Nginx. Приклад конфігурації:
  ```nginx
  server {
      listen 80;
      server_name yourdomain.com;
      root /path/to/MiroFish/frontend/dist;
      index index.html;
      location / {
          try_files $uri $uri/ /index.html;
      }
      location /api/ {
          proxy_pass http://localhost:5001/;
      }
  }
  ```

### Backend
- Запуск бекенду за допомогою `gunicorn` (для Python):
  ```bash
  uv run gunicorn -w 4 -b 0.0.0.0:5001 backend.app.main:app
  ```

## 4. Автономне розгортання (100% Offline)

Для закритих корпоративних мереж зверніться до документа `OFFLINE.md`. Основний підхід:
1. Попереднє завантаження всіх NPM та Python залежностей на машині з доступом до інтернету.
2. Використання локального Zep-сервера (Zep Community Edition) замість хмарного `zep-cloud`.
3. Використання локальних LLM-моделей (наприклад, через Ollama або LM Studio).

## 5. Масштабування (Multinode)

Для обробки симуляцій з мільйонами агентів:
- **База даних**: Розділіть Zep-сервер на окремі вузли.
- **Бекенд**: Запускайте кілька екземплярів бекенду за балансувальником навантаження (наприклад, Nginx або HAProxy).
- **LLM API**: Використовуйте рішення для кешування запитів або розподіленого доступу до пулу локальних LLM (наприклад, vLLM).
