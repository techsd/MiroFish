# Рецепти та приклади (Cookbook)

Цей документ містить практичні приклади використання системи MiroFish для вирішення конкретних завдань.

## Рецепт 1: Симуляція A/B тестування маркетингової кампанії

**Завдання:** Перевірити реакцію цільової аудиторії на два різні формати реклами нового продукту.

**Кроки:**
1. **Підготовка насіння реальності (Reality Seed):** 
   Створіть текстовий файл (`campaign_data.txt`), що містить опис обох рекламних кампаній, ціни, характеристики продукту та інформацію про бренд.
2. **Запуск MiroFish:**
   Завантажте файл у систему.
3. **Промпт симуляції:**
   `"Проаналізуй реакцію аудиторії на Кампанію А та Кампанію Б. Яка з них викликає більше позитивних відгуків серед молоді (MBTI: ENFP, INFP)? Який продукт вони схильні купити?"`
4. **Конфігурація (Tuning):**
   У меню налаштувань раундів виберіть 50 раундів симуляції (приблизно 2 години симульованого часу).
5. **Результат:** 
   Дочекайтеся генерації звіту Report Agent. Вивчіть графік дрейфу думок, щоб визначити кампанію-переможця.

## Рецепт 2: Інтеграція власної локальної моделі (Ollama)

**Завдання:** Знизити витрати на API, запустивши симуляцію повністю безкоштовно на локальному обладнанні.

**Кроки:**
1. Встановіть [Ollama](https://ollama.com/) на вашому комп'ютері.
2. Завантажте модель, наприклад `llama3`:
   ```bash
   ollama run llama3
   ```
3. Відкрийте файл `.env` у MiroFish і внесіть зміни:
   ```env
   LLM_BASE_URL=http://127.0.0.1:11434/v1
   LLM_MODEL_NAME=llama3
   LLM_API_KEY=ollama
   ```
4. Запустіть систему. Тепер усі запити, включно з внутрішньою логікою агентів, будуть оброблятися локально.

## Рецепт 3: Віртуальне інтерв'ю (Deep Interaction)

**Завдання:** Зрозуміти глибинні мотиви специфічного агента, який у симуляції проявив різко негативну реакцію.

**Кроки:**
1. Після завершення етапу "Генерація звіту" перейдіть на вкладку "Глибока взаємодія" (Step 5).
2. Знайдіть у списку агентів того, хто публікував негативні пости.
3. Оберіть цього агента для чату.
4. Почніть діалог:
   *Ви:* "Я помітив, що ви негативно відгукувалися про нове законодавство. Чому?"
   *Агент:* (Відповість, спираючись на згенеровану йому біографію, MBTI та пам'ять, що була сформована під час симуляції).
5. Використовуйте інструмент масового опитування (Survey), щоб надіслати це ж запитання всім агентам з подібними демографічними характеристиками.

## Рецепт 4: Адаптація параметрів часу

**Завдання:** Симулювати довгостроковий вплив події (наприклад, реакцію суспільства через тиждень).

**Кроки:**
1. Перейдіть до етапу "Налаштування середовища".
2. Знайдіть блок конфігурації часу.
3. Змініть параметр `minutesPerRound` на 1440 (1 день = 1 раунд).
4. Встановіть `rounds` = 7.
5. Запустіть симуляцію. Кожен раунд агенти генеруватимуть менше повідомлень, але загальний часовий проміжок покриє цілий тиждень, демонструючи зміну трендів з часом.
