# Гайд з переналаштування MiroFish (TUNE.md)

Цей посібник містить прості та чіткі інструкції щодо базової зміни конфігурації проекту MiroFish.

## 1. Налаштування провайдерів LLM (.env)

Проект підтримує будь-яке API, яке сумісне з форматом OpenAI SDK (наприклад: OpenAI, Anthropic через проксі, локальні моделі через LM Studio/Ollama, або хмарні китайські моделі).

Усі базові налаштування робляться у файлі `.env` у кореневій теці проекту:

### Стандартна конфігурація (приблизно)
```env
LLM_API_KEY=ваш_ключ_від_провайдера
LLM_BASE_URL=https://api.openai.com/v1  # Або інший сумісний endpoint
LLM_MODEL_NAME=gpt-4o-mini              # Назва моделі для стандартних завдань
```

### Конфігурація для прискорення (опціонально)
Для важких завдань симуляції ви можете вказати додаткову (швидшу або потужнішу) модель:
```env
LLM_BOOST_API_KEY=інший_ключ
LLM_BOOST_BASE_URL=інший_endpoint
LLM_BOOST_MODEL_NAME=gpt-4o
```
Якщо ви не хочете використовувати розділення моделей, просто видаліть ці рядки з `.env`.

## 2. Налаштування портів та мережі

За замовчуванням проект використовує порт 3000 для Frontend та порт 5001 для Backend.

### Зміна порту Backend
За замовчуванням бекенд запускається на порту 5001. Ви можете змінити це через змінні середовища (додавши їх у `.env` або задавши в терміналі):
```env
FLASK_HOST=0.0.0.0
FLASK_PORT=8080
```

Якщо ви змінюєте порт бекенду, вам ОБОВ'ЯЗКОВО потрібно оновити налаштування проксі у Frontend. Відкрийте файл `frontend/vite.config.js` та змініть `target`:
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:8080', // Змініть тут на ваш новий порт бекенду
    changeOrigin: true,
    secure: false
  }
}
```

### Зміна порту Frontend
Порт фронтенду жорстко прописаний у `frontend/vite.config.js`. Відкрийте цей файл і змініть параметр `port`:
```javascript
server: {
  port: 8000, // Ваш новий порт для веб-інтерфейсу
  open: true,
  // ... proxy config
}
```

### Зміна портів у Docker
Якщо ви використовуєте Docker, усі зміни портів можна зробити виключно у файлі `docker-compose.yml` в розділі `ports`:
```yaml
ports:
  - "8000:3000" # Зовнішній порт фронтенду 8000 -> внутрішній 3000
  - "8080:5001" # Зовнішній порт бекенду 8080 -> внутрішній 5001
```

## 3. Ліміти симуляцій OASIS (.env)

Щоб запобігти зайвим витратам API-токенів під час тестування, ви можете обмежити кількість раундів симуляції та роздумів агентів через `.env`:

```env
# Кількість раундів симуляції (стандартно 10)
OASIS_DEFAULT_MAX_ROUNDS=3

# Ліміти на роздуми агента звітності
REPORT_AGENT_MAX_TOOL_CALLS=3
REPORT_AGENT_MAX_REFLECTION_ROUNDS=1
```

## 4. Пам'ять агентів (Zep)
Проект використовує Zep Cloud для довгострокової пам'яті агентів. Ключ API обов'язковий і вказується у `.env`:
```env
ZEP_API_KEY=ваш_ключ_zep
```
Якщо Zep API змінить endpoint у майбутньому, або ви піднімете локальний Zep сервер, вам доведеться внести зміни безпосередньо у код в `backend/app/config.py` або додати обробку ZEP_API_URL. Наразі підтримується тільки хмарна версія через SDK.
