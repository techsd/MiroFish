## ⚡ Огляд

**MiroFish** — це система прогнозування нового покоління на базі мультиагентних технологій. Завантаживши звіт, новину чи статтю, рушій витягує "насіння реальності" та будує паралельний цифровий світ (на базі GraphRAG та Zep). Тисячі інтелектуальних агентів з унікальними характерами взаємодіють між собою, генеруючи еволюцію думок, яку ви можете спостерігати та аналізувати.

> **Що ви робите:** Завантажуєте PDF/MD/TXT та описуєте природною мовою, що саме хочете перевірити (наприклад: "Як відреагує спільнота на цей закон?").
> **Що робить MiroFish:** Повертає вам розгорнутий аналітичний звіт та надає доступ до живого цифрового світу для глибшої взаємодії (віртуальні інтерв'ю, опитування).

## 🛠️ Практичне Застосування та Функціонал

Завдяки гнучкій архітектурі, проект можна використовувати для:
1. **Аналізу громадської думки:** Прогнозування реакції соцмереж на вихід нового продукту, прес-релізу або законопроекту.
2. **Маркетингових досліджень:** Запуск "віртуальних інтерв'ю" з тисячами ШІ-користувачів для тестування ціноутворення чи позиціонування.
3. **Сценарного моделювання:** "Що було б, якби..." — перевірка альтернативних розвитків подій.

## 🚀 Швидкий Старт

Для запуску проекту вам знадобляться: Node.js (18+) та Python (3.11+). Для управління пакетами Python ми використовуємо сучасний швидкий менеджер `uv`.

### 1. Змінні середовища
Створіть файл `.env` (можна скопіювати з `.env.example`) та додайте ключі:
```env
LLM_API_KEY=ваш_ключ_від_llm
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus
ZEP_API_KEY=ваш_ключ_від_zep
```

### 2. Встановлення та запуск
```bash
# Встановлення всіх залежностей
npm run setup:all

# Запуск системи (Фронтенд: 3000, Бекенд: 5001)
npm run dev
```

## ⚙️ Тонке Налаштування (Tuning) та Оптимізація

Проект дуже гнучкий і дозволяє вам змінювати як LLM-провайдерів, так і архітектурні обмеження. Усі деталі налаштування зібрані у нашому [TUNE.md](./TUNE.md).

- **Зміна портів:** Якщо порти `3000` або `5001` зайняті, ви можете змінити їх у `vite.config.js` та файлі `.env`.
- **Локальні LLM:** MiroFish чудово працює з локальними моделями через LM Studio або Ollama. Достатньо змінити `LLM_BASE_URL` на `http://127.0.0.1:1234/v1` (детальніше див. `OFFLINE.md`).
- **Оптимізація токенів:** Якщо ви використовуєте потужні, але дорогі моделі (наприклад GPT-4o), рекомендуємо зменшити максимальну кількість раундів симуляції через змінну `MAX_SIMULATION_ROUNDS` або вибирати "Користувацькі налаштування" під час запуску через UI.

## 📴 Робота без Інтернету (100% Offline Режим)

Для закритих корпоративних мереж або максимальної приватності MiroFish можна розгорнути повністю автономно. Цей процес детально описаний у [OFFLINE.md](./OFFLINE.md).

**Короткий алгоритм переходу в Offline:**
1. Завантажте всі NPM та Python залежності на комп'ютері з інтернетом (`npm ci` та `uv sync`).
2. Використовуйте локальні моделі (LM Studio/Ollama).
3. **Zep Server:** Замість хмарного `zep-cloud` підніміть Zep Community Edition у Docker (`docker run -p 8000:8000 getzep/zep:latest`). 
4. Для автоматичної заміни залежностей `zep-cloud` на `zep-python` в коді скористайтеся Zero Config скриптом, описаним у `OFFLINE.md`.

## 🔄 Життєвий цикл симуляції

1. **Побудова Графа:** З вашого тексту формується граф знань (через Zep), що стає "глобальною пам'яттю" симульованого світу.
2. **Налаштування середовища:** Рушій генерує сотні персон, наділяючи їх унікальними біографіями, пам'яттю та MBTI профілем.
3. **Паралельна симуляція:** Агенти живуть у двох віртуальних платформах (аналоги Twitter та Reddit) — читають новини, створюють пости, сперечаються та формують тренди.
4. **Звітність:** Агент-аналітик (ReportAgent) перетравлює весь масив згенерованих даних за допомогою 4 спеціалізованих RAG-інструментів і формує фінальний markdown-документ.
5. **Глибока взаємодія:** Ви отримуєте доступ до консолі управління. Можете провести віртуальне інтерв'ю з будь-яким агентом або відправити масове опитування.
